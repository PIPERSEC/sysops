# Memory Bank

This directory serves as the persistent memory for AI assistants working on systems engineering projects.

## Purpose
The memory bank stores:
- Project-specific context and history
- Architectural decisions and rationale
- Lessons learned and best practices
- Common issues and resolutions
- Team preferences and standards

## Structure

### project-context.md
Current project state, goals, and active work streams

### architectural-decisions.md
Record of architectural decision records (ADRs) with context and reasoning

### best-practices.md
Accumulated knowledge and best practices specific to this environment

### common-issues.md
Known issues, gotchas, and their solutions

### preferences.md
Team coding standards, tool preferences, and conventions

## Usage
When starting a new conversation or project:
1. Review relevant memory bank files for context
2. Update files as new decisions are made
3. Document lessons learned after significant work
4. Keep information current and remove outdated content

## Maintenance
- Review monthly for accuracy
- Archive outdated information
- Consolidate duplicate information
- Ensure clarity and conciseness
